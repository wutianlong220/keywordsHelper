## 一、长尾词自动扩展

### 场景
用户输入核心词（如 `zerogpt`），skill 除了给出该词本身的分类（B 类），还**自动派生**一批高价值长尾词：

```
zerogpt           → 主词，B 类（中小品牌，导航意图）
zerogpt alternative → A 类（比较/替代品意图）
zerogpt free       → A 类（免费需求）
zerogpt review     → A 类（评测意图）
zerogpt vs xxx     → A 类（对比意图）
```

### 价值
- 节省用户手动想长尾词的时间
- 自动发现"主词打不进去、但长尾词有机会"的赛道
- 特别适合处理中小品牌词（主词 B 类 → 派生 A 类长尾）

### 实现思路
维护一个"高价值修饰词库"，常见修饰词：

| 类型 | 修饰词示例 |
|---|---|
| 比较型 | alternative, vs, compared to, substitute |
| 评测型 | review, ratings, pros and cons |
| 价格型 | free, cheap, discount, coupon, price |
| 排行型 | best, top 10, alternatives |
| 功能型 | online, download, no signup |
| 工具型 | calculator, generator, tracker, maker |

派生时，把每个修饰词和主词组合，生成新词条，让 AI 二次分类。

### 风险
1. **造词过多稀释注意力**：自动派生 10 个修饰词 × 100 个主词 = 1000 个新词条，可能让用户淹没
2. **大量派生词是垃圾**：很多组合在 SEO 上没人搜
3. **数据来源问题**：理想情况下应该用 Google Suggest / SEMrush 的真实搜索词来扩展，而不是凭空造词

### 触发条件（什么时候做这个功能）
- 用户跑了几次 Step 1 后，反馈"我每次都在手动想长尾词，太烦了"
- 此时再启动这个功能，并且：
  - 默认**关闭**，需要用户显式开启（避免污染输出）
  - 输出时**单独放在一个 Sheet**（如"派生长尾词"），跟主分类隔开

## 二、多 Excel 已收藏词的再次汇总处理

### 场景
用户每次跑关键词分类，都会生成一个新时间戳的 Excel（`关键词分类_YYYYMMDD_HHMM.xlsx`）。用户会在 Excel 的「收藏」列手工打勾。几次运行后，用户想：**把我最近几批里「收藏」过的所有词汇总到一个文件里**，作为"我的种子词库"长期保留。

### 价值
- 跨批次沉淀用户的精选词，避免"上次收藏的那批词不知道去哪了"
- 不用手动打开每个 Excel 复制粘贴
- 形成个人化的「高价值词库」，可作为未来 LLM 分类的优先级信号

### 实现思路
新增脚本 `scripts/merge_favorites.py`：

- 输入：`outputs/` 下用户指定的若干个 Excel 路径（或 `--all` 自动选所有）
- 遍历每个 Excel，提取「收藏」列（第 8 列）值为 `"收藏"` 的行
- 按关键词去重，输出到一个新 Excel：`outputs/收藏汇总_YYYYMMDD_HHMM.xlsx`
- 输出列：英文原文 / 来源 Excel / 类型 / 建议 / 目标用户 / intitle / SERP / 谷歌趋势 / Ahrefs KD
- 默认去重策略：保留**首次收藏**的版本；CLI 参数可选 `latest` / `keep_all`

### 风险
1. **跨批次判定可能不一致**：同关键词第一次被标"建议"且收藏，第二次被标"待研究"且未收藏 —— 合并时以哪个为准？
2. **去重策略争议**：用户可能想看"所有批次的全部收藏记录"而不是只取一条
3. **隐私泄露**：汇总文件包含历史批次的关键词与人工收藏选择，必须放在 `outputs/` 内（已入 `.gitignore`）
4. **数据漂移**：Excel 是用户的本地工作文件，可能被手工编辑过（修改「收藏」），汇总时拿到的"收藏"是编辑后的状态，不一定是脚本生成时的状态

### 触发条件（什么时候做这个功能）
- 用户明确反馈："我跑了好几批了，想知道我总共收藏了哪些词"
- 此时再启动这个功能，且：
  - CLI 参数显式选择 Excel 路径（默认不接受 `--all`，避免误把临时 Excel 收进来）
  - 汇总结果直接在对话里打印统计（"共汇总 N 个唯一词，来自 M 个 Excel"），Excel 路径 + 简表同步输出
