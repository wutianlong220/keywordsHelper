## 主题词提取与生态评估工具——设计思路

本文记录了一个自动化工具的完整设计思路：输入一个长尾关键词，自动提取核心主题词，并评估该主题词背后的生态是否值得深挖。

---

### 一、工具要解决什么问题

手动调研关键词时，我们经常遇到这种情况：一个长尾词本身不值得做（搜索量低、竞争激烈、变现困难），但它背后的主题词可能有一个庞大的关键词生态。手动从每个长尾词中提取主题词、再围绕主题词扫描生态，过程繁琐且耗时。这个工具的目标是自动化这个流程，快速判断一个长尾词背后的主题是否值得深挖。

---

### 二、核心流程

```
输入长尾关键词
    ↓
第一步：提取核心主题词
    ↓
第二步：围绕主题词扫描生态
    ↓
第三步：计算"生态评分"
    ↓
输出：建议深挖 / 建议观望 / 建议放弃
```

---

### 三、第一步：从长尾词提取主题词

**目标：** 去掉修饰词、品牌词、功能描述词，留下核心实体/主题。

**示例：**
- "nursery layout drawing" → 去掉 "layout"（功能描述）和 "drawing"（形式描述）→ 核心主题词："nursery"
- "govee smart nugget ice maker" → 去掉品牌 "govee" 和修饰词 "smart" → 核心主题词："nugget ice maker"
- "best running shoes for flat feet 2026" → 去掉 "best"、"for"、"2026" → 核心主题词："running shoes flat feet"
- "thunderstorm tracker uk" → 去掉地域 "uk" → 核心主题词："thunderstorm tracker"

**实现方式：**

**方案 A（简单，推荐初期使用）：** 维护一个常见修饰词/停用词列表，从关键词中移除这些词和已知品牌名，剩下的就是候选主题词。

需要过滤的词包括：
- 通用修饰词：best, top, free, cheap, affordable, premium, professional, easy, simple, quick, fast
- 时间词：2024, 2025, 2026, today, now, new, latest
- 功能/形式词：review, reviews, guide, tutorial, how to, what is, vs, comparison, ideas, tips, examples, template, design, layout, drawing, tracker, tool, app, software, online, near me
- 连接词：for, and, or, with, in, on, at, to, of, the, a, an
- 地域词（可选保留或移除）：uk, us, usa, australia, canada, india, philippines

同时维护一个品牌词库（如 govee, samsung, apple, nike 等），识别并移除品牌词。

**方案 B（高级）：** 接入 NLP 服务（如 OpenAI API），让 AI 分析关键词的核心主题。适合处理更复杂的情况（比如一个词有多种含义时），但每次调用有成本。

**建议初期用方案 A，简单高效。** 后续如果发现准确率不够，再考虑引入方案 B 作为补充。

---

### 四、第二步：围绕主题词扫描生态

**目标：** 找出这个主题词下面有多少相关关键词，以及这些关键词的质量如何。

**数据来源（由免费到付费排列）：**

**1. Google Autocomplete / Google Suggest（免费）**

当你在 Google 搜索框输入一个词时，Google 会给出联想词建议。这些联想词代表了真实的用户搜索行为。

- 可以直接输入主题词获取联想词
- 也可以在主题词后面逐个添加 a-z 字母来获取更全面的联想词（比如 "nursery a"、"nursery b"、"nursery c"...）
- 还可以尝试在主题词前面加修饰词（"best nursery..."、"how to nursery..."）

Python 库 `googlesearch-python` 或 `suggest-api` 可以批量获取这些数据。

**2. People Also Ask（PAA）和 Related Searches（免费，需抓取）**

搜索结果页中间的"其他人还在问"和底部的"相关搜索"，是 Google 认为跟你的主题词高度相关的内容。可以通过抓取 SERP 页面获取。

**3. Google Keyword Planner（免费，需 Google Ads 账户）**

如果有 Google Ads 账户，Keyword Planner 可以提供相关词和搜索量范围。免费但数据精度有限（给的是范围而不是精确数字）。

**4. 付费 API（DataForSEO / Semrush / Ahrefs）**

提供精确的搜索量、竞争度、CPC 等数据。最准确但有费用。DataForSEO 按调用量付费，比较灵活，适合初期小规模使用。

**建议初期方案：** Google Suggest + PAA + Related Searches 的免费方案，先把工具跑起来。等验证了实用性，再接入付费 API 提升数据精度。

---

### 五、第三步：计算生态评分

**核心逻辑：** 从四个维度评估主题词的生态健康度，加权合成一个 0-100 的评分。

**维度一：广度（关键词数量）—— 权重 30%**

围绕这个主题词，能挖掘出多少个相关关键词？

- 50+ 个相关词 → 满分
- 30-49 个 → 80 分
- 20-29 个 → 60 分
- 10-19 个 → 40 分
- 10 个以下 → 20 分

**维度二：深度（搜索量分布）—— 权重 30%**

这些相关词的搜索量是怎样的？（如果使用免费方案，可以用 Google Suggest 的联想词数量作为搜索量的间接指标——联想词越多，说明搜索量越大）

- 有 5+ 个高搜索量词（月搜索量 > 5000）→ 满分
- 有 3-4 个高搜索量词 → 80 分
- 有 1-2 个高搜索量词 → 60 分
- 全部是低搜索量词 → 30 分

**维度三：竞争空白度 —— 权重 25%**

抽查几个有代表性的相关词，看 intitle 结果数量。

- 大部分词 intitle 结果 < 100 → 竞争空白，满分
- 大部分词 intitle 结果 100-500 → 中等竞争，70 分
- 大部分词 intitle 结果 500-1000 → 竞争较激烈，40 分
- 大部分词 intitle 结果 > 1000 → 红海，20 分

**维度四：SERP 友好度 —— 权重 15%**

抽查 SERP 形态，判断是否对内容站/AdSense 友好。

- 以文字结果为主 → 满分
- 有图片或视频但不霸屏 → 70 分
- 图片/视频/购物卡片霸屏 → 30 分

**最终评分：** 四个维度按权重加权计算。

| 评分区间 | 建议 |
|---------|------|
| 80-100 | 根系非常发达，强烈建议深挖 |
| 50-79 | 有一定生态，值得进一步分析 |
| 20-49 | 生态薄弱，优先级低 |
| 0-19 | 基本没有生态，直接放弃 |

### 六、工具的技术架构

**最简版本（推荐先做这个）：**

一个 Python 脚本，输入一个关键词列表，对每个词执行完整流程，输出 CSV 表格。

```
输入：keywords.csv（一行一个关键词）
输出：results.csv

results.csv 字段：
- 原始关键词
- 提取的主题词
- 相关词数量
- 生态评分（0-100）
- 建议（深挖/观望/放弃）
- 相关词列表（前 20 个）
```

**进阶版本：**

加一个简单的 Web 界面（Flask 或 Streamlit），可以在浏览器里输入关键词、查看可视化的生态图谱和评分详情。

---

### 七、工具的局限性

这个工具能帮你加速筛选，但不能完全替代人工判断：

- 有些主题虽然关键词多，但全都是品牌锁定的（如 Govee），工具无法识别这种情况
- 有些主题虽然搜索量大，但内容制作成本极高（需要专业知识或视觉素材），工具无法评估
- SERP 形态的判断在免费方案下精度有限，可能需要人工抽查验证
- 工具给出的是初步评分和排序，最终的深挖决策还需要结合人工分析框架
