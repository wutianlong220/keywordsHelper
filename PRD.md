# PRD：关键词筛选助手

> 版本：v1.0
> 日期：2026-07-02
> 状态：待实施

---

## 1. Executive Summary

### Problem Statement

从多渠道（推特热搜、谷歌下拉词、Google Trends、SEMrush 等）收集的英文关键词数量庞大（几十到几百上千），其中混有大量无商业价值的垃圾词（娱乐梗、人名、世界级大品牌）。**人工逐个判断耗时巨大，且容易漏判、误判**。

### Proposed Solution

在 `/Users/xingchen/GitHub/keywordsHelper/` 项目里，做一个**对话式驱动的关键词批量分类工具**：用户在 Claude Code 里贴入关键词列表，工具调用 LLM 完成语义分类，并生成一个**带可点击链接（intitle/SERP/趋势/Ahrefs）和颜色标注的 Excel 表格**，作为后续人工精选（Step 2~5）的起点。

### Success Criteria

| # | 指标 | 目标 |
|---|---|---|
| 1 | **批量处理能力** | 单次输入 100~500 个关键词能正常跑完 |
| 2 | **运行时长** | 100 词 ≤ 5 分钟产出 Excel |
| 3 | **分类质量（人工抽检）** | 用户抽检 10 个词，主观判定分类合理率 ≥ 80% |
| 4 | **Excel 规格符合度** | 列定义、颜色、4 类链接、冻结首行、筛选器 100% 符合本 PRD 第 4.2 节 |
| 5 | **链接可点击率** | 4 列链接 100% 可点击、URL 编码正确、目标页面与意图一致 |

> 验收方式：用户人工看一眼 + 抽检几个词判断，**不做定量评估**。

## 2. User Experience & Functionality

### 2.1 User Personas

**唯一用户**：项目所有者（星辰），独立做英文 SEO 站点的开发者，长期从多渠道收集关键词、需要快速筛出有商业价值的种子词。

### 2.2 User Stories

#### Story 1：批量分类
> As a 站长，我想要把几百个关键词一次性丢给工具，让 AI 帮我把它们分成 A/B/C/待定，这样我不用逐个手动判断。

**Acceptance Criteria**：
- ✅ 支持输入 1~500 个关键词
- ✅ 每个关键词输出 7 个字段（见 4.2 节）
- ✅ 品牌库命中的词自动标 C 并溯源 `[品牌库:xxx]`
- ✅ 未命中的词由 LLM 分类
- ✅ 输出单 Sheet Excel 文件

#### Story 2：可点击的二次检查链接
> As a 站长，我想在 Excel 里直接点击每个关键词对应的 intitle/SERP/趋势/Ahrefs 链接，省去我手工去每个网站搜。

**Acceptance Criteria**：
- ✅ 4 列链接，每列 URL 模板见 4.3 节
- ✅ 关键词正确 URL 编码
- ✅ 备注信息不混入 URL
- ✅ 4 列用 4 种颜色区分（蓝/紫/橙/绿）

#### Story 3：品牌库可持续积累
> As a 站长，我跑了几次后发现某些词（品牌、垃圾词）反复出现且无用，我想把它们永久加入过滤库，下次自动归 C。

**Acceptance Criteria**：
- ✅ `brands.txt` 支持手动编辑（一行一个品牌）
- ✅ 子串匹配（大小写不敏感）
- ✅ 注释行（`#` 开头）和空行忽略
- ✅ 修改后下次运行立即生效

### 2.3 Non-Goals（明确不做的范围）

| 不做的功能 | 理由 |
|---|---|
| 全局 skill 安装 | 已决定做项目，不装 `~/.claude/skills/` |
| Step 2~5 的自动化（intitle 真实查询、趋势数据抓取、SERP 分析） | 当前只做 Step 1，后续步骤由用户点击链接人工完成 |
| 黑名单词库（非品牌的屏蔽词） | 当前不做，未来按需追加 |
| 长尾词自动扩展 | 已记录在 `待优化指南.md`，未来按需做 |
| SEMrush API 集成 | 用户当前没有 API key |
| 定量分类质量评估 | 用户明确选择"看一眼就行"，不做指标化评估 |
| 3 Sheet 物理拆分 | 改为单 Sheet + 筛选器，更灵活 |
| 子串匹配的精确豁免机制（如 `=apple` 表示精确匹配） | 当前不做，依靠 `[品牌库:xxx]` 溯源标签发现误伤 |

## 3. AI System Requirements

### 3.1 Tool Requirements

| 工具 | 用途 | 来源 |
|---|---|---|
| Claude（Claude Code 内置） | 关键词语义分类 | 对话式直接调用 |
| Python 3 | 数据处理与 Excel 生成 | 系统自带 |
| `openpyxl` 库 | 生成 .xlsx 文件 | `pip install openpyxl` |

### 3.2 LLM 分类 Prompt 设计原则

> 当前阶段不加复杂判定细节，**用最简 prompt 先跑一版**，看效果再迭代。

**最小 prompt 结构**（v1，已升级为 v1.1 双轴）：

```
你是关键词分类专家。对每个英文关键词输出 JSON，字段：
- translation: 中文直译
- intent: 搜索意图（信息型 / 导航型 / 交易型 / 工具型）
- type: 类型（工具 / 内容 / 带货 / 游戏 / 导航 / 空）—— 关键词天然适合做的网站类型
- monetization: 建议变现路径（多选，从 [SaaS, AdSense, 联盟, 电商, 导航, 待定] 中选）
- advice: 给用户的行动建议（建议 / 待研究 / 跳过）
- target_user: 谁会搜这个关键词（必须写真实人群描述；拆不出时填「无法判定」，禁止写「歧义意图」「未知」「不确定」等元评论）

类型判定规则：
- 工具：tracker / extractor / converter / generator / uploader / planner / keyboard 等工具化后缀
- 内容：layout / pattern / specs / recipe / tips 等信息型长尾
- 带货：品牌名 + 产品修饰（buy / review / best / sale），适合联盟/电商页
- 游戏：游戏名 / tier list / codes / skins / walkthrough
- 导航：平台名（搜索意图直达已有站点）
- 空：以上都不属于（影视 / 单次事件 / 歧义查询 / 本地商家 / 娱乐梗）

建议判定规则：
- 建议：明确建得起来，关键词特征清晰指向可落地项目
- 待研究：可能有用但需查 SERP/intitle 确认竞争或搜索量
- 跳过：影视 / 单次事件 / 纯娱乐梗 / 品牌导航 / 太宽泛 / 无商业意图 / 已知大平台

用户备注（如有）必须重视，作为上下文判断。
```

### 3.3 批处理策略

- **批大小**：每批 50 个关键词送 LLM
- **重试机制**：JSON 解析失败时重试 1 次，仍失败则该词标记 `[LLM 解析失败]`，不阻塞整体流程
- **并发**：v1 不并发，串行处理（简单可靠）

### 3.4 Evaluation Strategy

**当前阶段**：用户人工抽检，无定量指标。

**未来可能扩展**（不属 v1 范围）：
- 维护一个标注好的测试集（30~50 词）
- 跑分类后计算 A/B/C 各类准确率
- 每次调整 prompt 后回归

## 4. Technical Specifications

### 4.1 Architecture Overview

```
[用户在 Claude Code 里输入]
   "对这批关键词分类：xxx / xxx / xxx"
   或："处理 inputs/keywords-07-02.txt"
        │
        ▼
[解析输入] ─── 一行一个，识别括号备注并丢弃
        │
        ▼
[读取 brands.txt] ─── 过滤注释/空行，转小写
        │
        ▼
[品牌库子串匹配] ─── 命中 → 标 C 类，note 加 [品牌库:xxx]
        │              未命中 → 进 LLM 队列
        ▼
[LLM 批量分类] ─── 50 词/批，串行调用
        │
        ▼
[合并结果] ─── 品牌库命中 + LLM 分类
        │
        ▼
[Python + openpyxl 生成 xlsx]
   - 单 Sheet + 冻结首行 + 筛选器
   - 行按分类着色
   - 4 列链接按列着色
        │
        ▼
[输出文件]
   outputs/关键词分类_YYYYMMDD_HHMM.xlsx
        │
        ▼
[在对话里返回路径 + 简要统计]
   "已生成 Excel：xxx.xlsx
    共 100 词：A 类 20、B 类 35、待定 5、C 类 40（其中品牌库命中 15）"
```

### 4.2 Excel 表结构

**单 Sheet 表，列顺序固定**（v1.1 升级：双轴分类，详见附录 C）：

| # | 列名 | 内容 | 颜色 |
|---|---|---|---|
| 1 | 英文原文 | 用户原始输入的关键词部分（不含备注） | 按建议着色（见下） |
| 2 | 中文翻译 | LLM 直译，品牌库命中时填 `[品牌库]` | 同上 |
| 3 | 搜索意图 | 信息型 / 导航型 / 交易型 / 工具型 | 同上 |
| 4 | **类型** | 工具 / 内容 / 带货 / 游戏 / 导航 / （空）—— 关键词天然适合做的网站类型 | 同上 |
| 5 | 变现路径 | 多选，` / ` 分隔 | 同上 |
| 6 | **建议** | 建议 / 待研究 / 跳过 —— 给用户的行动建议 | 同上 |
| 7 | 目标用户 | 谁会搜这个关键词；品牌库命中加 `[品牌库:xxx]` 前缀 | 同上 |
| 8 | 收藏 | 下拉框选项：**收藏 / 丢弃 / 红利**（v1.3 起固定深绿 #548235 + 白字加粗；v1.5 起新增"丢弃"选项；v1.6 起新增"红利"选项，标记"流量红利词"——必须马上做、否则过期；toggle 仅决定文字内容，不切换背景色）。三者都与「建议」列不联动，由用户手动区分 | 深绿 #548235 + 白字 |
| 9 | **我的调研结论** | 自由文本，调研后手填备注（默认空，无下拉框、无 hyperlink）| 按建议着色（与左 7 列同色） |
| 10 | intitle 检查 | 可点击 URL | **浅蓝**（标头深蓝） |
| 11 | SERP 检查 | 可点击 URL | **浅紫**（标头深紫） |
| 12 | 谷歌趋势 | 可点击 URL | **浅橙**（标头深橙） |
| 13 | Ahrefs KD | 可点击 URL | **浅绿**（标头深绿） |

**行级着色**（按「建议」列决定，覆盖第 1~9 列，含 v1.4 新增的"我的调研结论"列；不覆盖收藏列 8 与链接列 10~13）：
- 建议 → 浅绿
- 待研究 → 浅黄
- 跳过 → 浅灰

**其他格式**：
- 首行冻结
- 第1列和第2列冻结
- 首行加筛选器
- 列宽自适应

### 4.3 URL 模板（关键词 URL 编码后替换 `{kw}`）

| 列 | 模板 |
|---|---|
| intitle 检查 | `https://www.google.com/search?q=intitle:%22{kw}%22&hl=en&gl=us` |
| SERP 检查 | `https://www.google.com/search?q={kw}&hl=en&gl=us` |
| 谷歌趋势（30 天） | `https://trends.google.com/trends/explore?q={kw}&geo=US&date=today%201-m` |
| Ahrefs KD | `https://ahrefs.com/keyword-difficulty/?country=us&input={kw}` |

**URL 编码规则**：
- 空格 → `%20`
- 双引号 → `%22`
- 其他特殊字符按 RFC 3986 标准 `urllib.parse.quote` 处理

### 4.4 文件结构

```
/Users/xingchen/GitHub/keywordsHelper/
├─ 关键词筛选完整流程.md     # 原始流程文档（已有）
├─ PRD.md                    # 本文档
├─ 待优化指南.md             # 未来功能备忘（已有）
├─ brands.txt                # 品牌词库（已有，可持续编辑）
├─ inputs/                   # 用户输入关键词的可选暂存目录
│  └─ keywords-*.txt
├─ outputs/                  # Excel 输出目录（自动创建）
│  └─ 关键词分类_YYYYMMDD_HHMM.xlsx
└─ scripts/                  # Python 脚本目录（v1 主要逻辑）
   ├─ generate_excel.py      # 接收分类结果 JSON，生成 xlsx
   └─ url_builder.py         # URL 模板拼接与编码
```

> 注：**对话式入口**意味着 Claude 在对话里直接读输入、调 LLM、生成中间 JSON，最后调用 `generate_excel.py` 出 Excel。脚本只是"出 Excel"这一步独立出来，方便复用与单测。

### 4.5 输入格式

**两种支持**：

1. **直接在对话里粘贴**：
   ```
   zerogpt
   thunderstorm tracker uk
   pag ibig loan tracker
   jmail.world （备注：推特热搜，爱泼斯坦案档案站）
   ```

2. **引用文件路径**：
   ```
   处理 inputs/keywords-07-02.txt
   ```

**解析规则**：
- 一行一个关键词
- 括号备注（中文括号 `（）` 或英文括号 `()` 内的内容）→ 解析后**丢弃**，不参与 URL 拼接和分类
- 空行忽略
- 重复关键词去重（保留首次出现的位置）

### 4.6 输出文件命名

```
outputs/关键词分类_YYYYMMDD_HHMM.xlsx
```

- 时间戳取**运行开始时间**
- 同一分钟内多次运行，文件名冲突时追加 `_2`、`_3`

### 4.7 Security & Privacy

- 关键词数据**只在本地处理**，不发送到任何第三方（除 Claude API 本身）
- 不记录用户关键词到任何日志文件
- Excel 输出在本地项目目录，用户自己管理

## 5. Risks & Roadmap

### 5.1 Phased Rollout

| 阶段 | 范围 | 状态 |
|---|---|---|
| **MVP（v1.0）** | Step 1 完整流程：品牌库匹配 + LLM 分类 + Excel 生成（含 4 列链接） | 当前 PRD 范围 |
| v1.4 | 新增"我的调研结论"列（自由文本，调研后手填） | 已实现 2026-07-03 |
| v1.5 | 收藏列下拉新增"丢弃"选项（与"收藏"并列，两者手动独立，不与"建议"列联动） | 已实现 2026-07-03 |
| v1.1 | 增加黑名单词库 `blacklist.txt`（非品牌屏蔽词） | 待用户提需求 |
| v1.2 | 调优 LLM prompt（基于实际误判案例） | MVP 上线后迭代 |
| v2.0 | 长尾词自动扩展 | 见 `待优化指南.md` |

### 5.2 Technical Risks

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| LLM 返回 JSON 格式错误 | 部分词分类失败 | 重试 1 次；仍失败标记 `[LLM 解析失败]`，不阻塞 |
| 品牌库子串匹配误伤（如 `apple` 命中 `pineapple`） | 合规词被误标 C | 一句话说明自动加 `[品牌库:xxx]` 溯源；用户筛出后从 `brands.txt` 删除即可 |
| 大批量（500 词）LLM 调用耗时长 | 用户体验差 | 50 词/批串行；提示用户耐心等待；进度可见 |
| `openpyxl` 未安装 | Excel 生成失败 | 项目首次运行时引导安装：`pip install openpyxl` |
| LLM 分类主观性强，不同批次可能不稳定 | 同一词不同次跑结果不一致 | MVP 阶段用户接受；v1.2 用 prompt 调优 + temperature=0 缓解 |
