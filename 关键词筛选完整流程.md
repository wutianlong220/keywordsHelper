# 关键词筛选完整流程（修订版）

从多渠道收集的多个英文关键词（数量从几十个到几百上千个不等）中，高效筛出有商业价值的种子关键词。

---

## 一、你（用户）需要准备什么

### 输入格式

只需把英文关键词原样给我，不需要附带任何数据（搜索量、KD 等都不需要）。

```
zerogpt
thunderstorm tracker uk
pag ibig loan tracker
```

### 什么时候加备注

**只有一种情况需要加备注**：这个词如果脱离上下文、光看字面，连你自己都觉得它像垃圾——那就加括号备注。比如：

```
jmail.world （备注：推特热搜，爱泼斯坦案档案站，大量用户分享此域名）
galatea （备注：推上新上架AI伴聊App，讨论度高）
```

备注一句话说清三件事：**在哪看到的 + 为什么火 + 用户动机。** 其他情况不需要备注。

## 二、完整流程案例（5步）

```
500个原始词
    │
    ▼
Step 1：AI批量分类（产出Excel）
    │  C类自动归档 ~200个，无需你审查
    │  A类+B类+待定 ~300个，你需要审查
    │
    ▼
Step 2：中文扫描 + intitle蓝海检查（合并执行）
    │  你边看中文释义，边顺手查 intitle:"关键词"
    │  intitle结果 < 1万 → 蓝海标记
    │  intitle结果 > 10万 → 红海，砍掉
    │  300 → 80
    │
    ▼
Step 3：谷歌趋势方向核验
    │  趋势上升？下降？有无飙升相关词？
    │  剔除趋势明显下降的词
    │  80 → 50
    │
    ▼
Step 4：SEMrush数据验证 + 评分排序
    │  批量查Volume、KD、CPC
    │  公式：CPC × (1-KD/100) × LOG10(Volume)
    │  从高到低排序，取Top 20
    │  50 → 20
    │
    ▼
Step 5：SERP人工深挖 + 竞品分析
    │  看前十名能不能打过
    │  下拉词、People Also Ask
    │  判断工具化机会
    │  20 → 10~30
    │
    ▼
最终种子关键词（10~30个）
```

## 三、每一步的详细说明

### Step 1：AI批量分类

**AI 做的事：** 对每个词输出以下信息——

| 列 | 说明 |
|---|---|
| 英文原文 | 原始关键词 |
| 中文翻译 | 直译，供你快速扫描 |
| 搜索意图 | 信息型 / 导航型 / 交易型 |
| 商业价值 | 高 / 中 / 低 / 待定 |
| 工具站适配 | 是 / 否 / 待定 |
| 一句话说明 | AI 的判断依据和备注 |

**分类逻辑：**
- **C 类**（直接归档）：纯娱乐梗词、人名词、品牌词、无商业意图、太宽泛无法定位意图 —— 自动归档到 Sheet3
- **B 类**（需你审查）：有可能有商业价值但不确定，或者需要更多上下文的词
- **A 类**（疑似有价值）：有明显商业意图或工具化潜力的词
- **待定**：AI 自己也看不准的词，标记出来让你特别留意

**交付物：** 一个 .xlsx 文件，3 个 Sheet（A类 / B类+待定 / C类已归档），含颜色标注、冻结首行、留空"你的决定"列。

**你的工作：** 重点看 B 类和待定词，判断升 A 还是降 C。A 类默认通过。C 类不需要逐词看。

---

### Step 2：中文扫描 + intitle蓝海检查（合并执行）

**为什么合并：** 当你在 Excel 里看到某个"高商业价值 + 工具站适配"的词时，顺手打开谷歌搜 `intitle:"关键词"`，看左上角结果数。一个词 3 秒，80 个词 5 分钟搞定。

**intitle 判断标准：**
- `<1 万` → 蓝海，标记进入下一步
- `1万~10万` → 灰色地带，看趋势再决定
- `>10 万` → 红海，砍掉

**为什么 intitle 在 SEMrush 之前：** 免费、无次数限制、1 秒一个词，反映的是"已经有人用这个词做标题的页面数量"——这是最直接的竞争信号。

---

### Step 3：谷歌趋势方向核验

对 Step 2 剩下的词逐个丢进谷歌趋势，只看两件事：

1. **趋势方向**：过去 12~24 个月是上升还是下降？明显下降的直接砍掉。
2. **飙升相关词**：谷歌趋势底部的"相关查询"里，如果有飙升关键词，那是额外收获，加回来。

**为什么这步重要：** 一个 SEMrush 显示搜索量 3000 的词，如果趋势在往下走，你花半年把网站做上去的时候流量可能已经腰斩了。反之，趋势上升的词是顺风行船。

---

### Step 4：SEMrush数据验证 + 评分排序

此时只剩 30~50 个词，直接批量丢 SEMrush 的 Keyword Overview。

**评分公式（在 Excel 里拉一列）：**

```
综合分 = CPC × (1 - KD ÷ 100) × LOG10(月搜索量)
```

**含义：**
- CPC 越高 → 商业价值越大
- KD 越低 → 越容易做上去
- 搜索量用对数 → 避免 120 万搜索量的词因为数量碾压而排在 5000 搜索量但超高 CPC 的词前面

**排序后取 Top 15~20 进入最后一步。**

---

### Step 5：SERP人工深挖 + 竞品分析

对最后 15~20 个词，人工在谷歌搜索：

- **看 SERP 前十名**：有没有大型权威站（Wikipedia、政府网站）？如果有，你能不能做出比这些更好的内容？
- **看广告**：有人在投广告吗？有广告 = 有人愿意为这个词花钱。
- **下拉词 & People Also Ask**：有没有意外的长尾词可以一起做？
- **拿排名前 3 的网站丢 SEMrush**：看它们从哪里获得流量，有没有内容空白你可以填。

**最终判断标准：**
- "工具化"机会 → 能不能做成在线工具/计算器/生成器？这是变现效率最高的方式。
- 内容站 → 能不能写出比 SERP 当前内容更全面的文章？
- 商业模型 → AdSense 还是联盟营销还是 SaaS？

## 四、商业意图判断框架

| 优先级 | 类型 | 特征 | 例子 | 为什么 |
|--------|------|------|------|--------|
| 🔴 最优先 | 工具型 | 用户想用某个工具/计算器 | `qr code generator` | 做一个在线工具，AdSense + 付费版双变现 |
| 🟡 次优先 | 比较/替代品型 | 用户在比价/找替代品 | `scrapingbee alternative` | 做对比站，联盟佣金变现 |
| 🟢 谨慎 | 纯信息型 | 用户想学/了解某个东西 | `astronomy picture of the day` | 流量可能大，但变现路径不直接 |
| ⚫ 丢弃 | 娱乐/梗/人名 | 用户消遣/无商业意图 | `kool aid man` | 怎么做都赚不到钱 |

## 七、核心原则

1. **先筛后挖**：深挖成本太高，不值得用在每一个词上。先用快筛把垃圾清掉。
2. **数据后置**：AI 分类（语义）→ intitle（免费竞争信号）→ 谷歌趋势（方向）→ SEMrush（精确数据），成本从低到高。
3. **你拍板**：AI 做分类建议，你做最终决定。Step 1 的输出不是终点，是你的起点。
4. **来源重于数据**：收集阶段记来源比记数据重要。一个"谁在聊这个词"的上下文，比 SEMrush 数据更能帮你做商业判断。
